DashCam Setup Guide

One clean path from zip file to running DashCam.

This guide is written for someone who is not comfortable with terminals or package managers. Follow the steps in order, and use the repair launcher if the first start shows warnings.

Double-click install
Double-click start
Use repair start if needed
Right Shift opens the menu

Onboarding

  1. Unzip the folder, then open the new DashCam_KoH folder.
  2. Double-click install.bat.
  3. After install finishes, double-click START_DASHCAM.bat.
  4. A terminal window should open. That is normal.
  5. Your output may look like Example 1 below. That is not a bad thing.
  6. Hold Ctrl + C for a few seconds to stop the terminal.
  7. The terminal may print a bunch of lines while stopping. That is good. You stopped it correctly. The window may also auto-close.
  8. Now double-click START_DASHCAM_AUDIT_FIX.bat.
  9. Wait until you see a local link in the terminal like the one in Example 2.
  10. Hover the link, hold Ctrl, and click it to open DashCam in your browser.
  11. Press the Right Shift key to show or hide the DashCam menu.

What The Files Do

install.bat installs the packages DashCam needs.

START_DASHCAM.bat starts DashCam normally.

START_DASHCAM_AUDIT_FIX.bat runs a package repair pass, then starts DashCam.

If you see warnings about vulnerabilities during install, that is exactly why the repair launcher exists.

Example 1

A normal first run may show install output and even a port change.

The repair launcher runs install, applies the audit fix, and starts DashCam.

Short Docs

Normal startup

Use START_DASHCAM.bat for everyday use.

Repair startup

Use START_DASHCAM_AUDIT_FIX.bat if the normal start looks unhealthy or you want to clear audit issues before launching.

Browser launch

Open the local link shown in the terminal. Hold Ctrl and click it.

Menu toggle

The Right Shift key opens and hides the in-app menu.

FAQ

Is the terminal window normal?
Yes. Leave it open while DashCam is running.
I see lots of lines. Did something break?
Usually no. Most of the output is normal install or startup logging.
The port changed from 5173 to something else. Is that bad?
No. Just open the local link the terminal gives you.
The terminal closed by itself. What now?
Run the repair launcher next. If it still closes immediately, check whether Node.js is installed.
How do I open the menu?
Press the Right Shift key.
